2.1. Какие типы исключений могут возникать в C# калькуляторе?
- DivideByZeroException — при делении на ноль.
- OverflowException — при выходе результата за пределы типа (например, для целых типов).
- FormatException — при некорректном вводе данных (если парсим строку в число).
- ArgumentException / ArgumentOutOfRangeException — при недопустимых аргументах (например, отрицательное число там, где оно не разрешено).
- InvalidOperationException — при некорректном состоянии операции.

2.2. Как бы вы организовали unit testing для этого проекта?
Я бы использовал один из фреймворков тестирования: xUnit, NUnit или MSTest.
Создал бы отдельный проект тестов (например, CalculatorApp.Tests) и написал тесты для каждого метода калькулятора:
- Нормальные случаи (положительные и отрицательные числа, ноль).
- Граничные случаи (очень большие / маленькие значения).
- Тесты на исключения (например, проверка, что при делении на ноль выбрасывается нужное исключение или возвращается корректное сообщение).


3. Workflow
3.1. Опишите идеальный процесс исправления бага в C# проекте через GitHub.
- Обнаружение бага и создание Issue с описанием, шагами воспроизведения и ожидаемым поведением.
- Назначение Issue на разработчика, добавление меток (bug, priority и т.д.).
- Создание ветки от main, связанной с Issue (например, bugfix/divide-by-zero).
- Внесение изменений в код, коммиты с понятными сообщениями и ссылкой на Issue.
- Запуск тестов, проверка, что баг исправлен и ничего не сломано.
- Создание Pull Request, код-ревью.
- Мёрдж ветки в main, закрытие Issue.

3.2. Какие метки наиболее полезны для C# проектов?
- `bug` — ошибки.
- `enhancement` / `feature-request` — новые функции.
- `refactoring` — улучшение структуры кода.
- `documentation` — задачи по документации.
- `in-progress`, `ready-for-review`, `help-wanted` — для состояния задачи.
- По желанию: `priority-high`, `priority-low`, `csharp`, `tests` и т.д.


4. Документирование
4.1. Как бы вы добавили XML documentation в C# код?
В C# можно включить генерацию XML-документации в настройках проекта и добавить к методам специальные комментарии вида.


4.2. Какая информация должна быть в README для C# проекта?

- Краткое описание проекта (что делает калькулятор).
- Требования к среде (версия .NET, например .NET 6.0).
- Инструкция по запуску (как собрать и запустить через командную строку или IDE).
- Примеры использования (какие операции поддерживаются).
- Информация о тестах (как запустить unit-тесты, если они есть).
- При необходимости — лицензия и контакты автора.
